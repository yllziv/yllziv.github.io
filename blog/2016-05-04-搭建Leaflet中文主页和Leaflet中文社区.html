发表于 2016-05-04   |   分类于 gis   |   阅读次数

webgis主流库对比

专业webgis库

arcgis for javascript（商用） 和 openlayers（开源）

  • 缺点：体积上较大：arcgis > ol3，学习路线陡峭。arcgis采用dojo框架，功能：arcgis>ol3。
  • 优点：地图编辑功能与分析功能强大：arcgis > ol3。ol3可定制api，将体积压缩到很小，并且可以学习源码。

简单webgis库

leaflet 和 mapbox（属于leaflet的扩展，均开源）

  • 缺点：地图编辑与分析功能较弱但也基本够用， 只支持WGS 1984 Web Mercator (Auxiliary Sphere)坐标系（Google Map，Bing Map，ArcGIS Online等服务支持的坐标系）， 其他坐标系处理较为麻烦，需要通过 Proj4库进行转换。mapbox版本2.0以上需要accessToken，不能离线使用。
  • 优点：体积小，适合手机端，api简洁。mapbox有一套完整的地图解决方案，重点在自定义地图上（配图）。

公众webgis库

百度地图、谷歌地图、高德地图、天地图、搜狗地图等。
天地图的api直接分析出请求接口，可以调用路径、api查询等等功能，通过抓包工具分析。

移动端webgis库（重点在于地图的缓存）

  • 使用hybird，通过android中的webview嵌入html，太费流量，加载慢。
  • 使用 ArcGIS Runtime SDK for iOS 和 ArcGIS Runtime SDK for Android 地图分析功能强大（GIS），离线使用。
  • 使用 Mapbox Android SDK 和 Mapbox iOS SDK 自定义地图（需要accessToken）
  • 使用百度地图、高德地图、谷歌地图、天地图等商用地图，在线使用，提供公共服务（需要accessToken）。

搭建Leaflet中文主页

缘起

在使用Leaflet的过程中，发现Leaflet的中文资料特别少，同时Leaflet在Github中的星是最多，在国外最流行。于是就想建立Leaflet中文主页，把Leaflet官方首页翻译过来，为国内学习Leaflet的人提供一个中文环境。这也是多年从开源社区中拿来主义，想回馈一下。说干就干！

Leaflet中文主页搭建

通过把Leaflet官网网站fork到自己的git仓库，然后对其中的内容进行翻译，最后部署到阿里云服务器上，配置Nginx虚拟主机作为网页容器。在万网上买了域名leafletjs.com.cn，过一段时间备案通过后即可上线，现在还是通过ip地址访问。

参与翻译

概述、教程、下载、插件四部分已经完成完成，文档部分内容太多，只翻译了一小部分。要是有人想一起翻译，可以Frok我Github中的Leaflet，然后clone到本地，执行下面命令：

Leaflet中文社区

准备

在搭建好Leaflet中文主页后，对比其他开源库主页，发现还缺少一个中文社区，来供大家交流问题。看到React Native中文社区做的很成功，查看源码发现用的是NodeBB搭建的社区。然后上NodeBB官网研究一番，哎哟，不错！就用它来搭建Leaflet中文社区。
